# Compaction + Cache Agent Cost Plan

## Goal
Add an optional agent-session simulation mode that models compaction and cached input more realistically, while keeping the current provider-based estimate as the baseline.

The new mode should let the user:
- keep the existing linear estimate untouched when the toggle is off
- switch on an agent-run scenario with compaction
- tune input/output cost allocation, including an `80/20` preset
- apply a cache discount to repeated input tokens

## Recommended approach
Use one pure helper for the session math and keep `app/page-client.tsx` as the UI/controller.

The helper should simulate turns instead of multiplying a single turn estimate by `agentTurns`. That gives us a place to model:
- repeated-context cache discounts
- compaction events that shrink history
- a realistic per-turn cost trail
- an adjustable input/output share for scenario mode

## Math model
Base pricing stays provider-driven:
- `inputCost = billableInputTokens / 1000 * inputRate`
- `outputCost = billableOutputTokens / 1000 * outputRate`

Session simulation should track these inputs:
- `promptTokens`
- `plannedOutputTokens`
- `turns`
- `cacheHitRate` or `cachedInputDiscount`
- `compactionThresholdPct`
- `compactionRetentionPct`
- `summaryOverheadTokens`
- `targetInputShare` and `targetOutputShare` for scenario mode

Turn flow:
1. Start with the prompt tokens and any fixed overhead tokens.
2. For each turn, compute fresh input, repeated input, and output tokens.
3. Apply cache discount only to repeated input.
4. If the accumulated context crosses the compaction threshold, add a compaction event that bills a summarization pass and reduces retained history.
5. Sum all turn costs and compaction costs into session totals.

Scenario split:
- In baseline mode, keep the existing provider-rate cost breakdown.
- In scenario mode, let the UI rebalance the displayed input/output cost share toward the selected target, with `80/20` as the default preset.
- Keep the total cost driven by the token simulation so the scenario remains anchored to actual usage shape.

## Files to change
- `app/page-client.tsx` for the new toggle, controls, receipt rows, and wiring
- `lib/sessionMath.ts` for the pure session simulator
- `README.md` for the new scenario mode and assumptions
- `lib/pricingParser.ts` only if we later add explicit cache pricing fields to the dataset

## Accuracy improvements to include
- show the assumptions used in the receipt
- expose compaction threshold and retention as tunable values
- expose cache discount separately from the output/input share target
- keep the baseline estimate visible so the scenario mode is clearly an alternate view, not a replacement for provider pricing
- show compaction count and cached-token count in the detail view

  - scenario toggle on/off
  - `80/20` preset
  - cache discount changing repeated-input cost
